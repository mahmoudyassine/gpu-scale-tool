# GPUscale.net share-link payload (`gpuscale.net/5`)

Read this when you need to hand-edit a decoded payload, debug a link, or
understand exactly what the encoder emits. For normal encoding, the friendly
spec in SKILL.md is all you need - `scripts/gpuscale_url.py` builds this
structure for you.

## URL anatomy

```
https://mahmoudyassine.github.io/gpuscale/#p=z:<fragment>
```

- `z:` fragment = base64url( deflate-raw( minified JSON payload ) ), no padding.
- `j:` fragment = base64url( plain JSON ) - the app's uncompressed fallback;
  the decoder accepts it, the encoder always emits `z:`.
- Everything lives in the URL fragment, so the project never touches a server;
  any base URL hosting the app works (gpuscale.net, the GitHub Pages mirror,
  or a local file).
- Python codec: encode with `zlib.compressobj(9, zlib.DEFLATED, -15)` then
  base64url-strip-padding; decode with `zlib.decompress(raw, -15)`. Valid
  fragments contain only `A-Z a-z 0-9 - _` - any other character (space, `+`,
  `/`, `=`, smart quote) means the link was mangled in transit.
- The app autosaves projects to the viewer's browser localStorage keyed by
  `projectId`. Opening a shared link whose `projectId` matches an existing
  local project can overwrite that local copy - when rebuilding a variant of
  someone's existing project, let the encoder mint a fresh `projectId`
  (default behavior) unless they explicitly want to replace theirs.

## Payload structure

```jsonc
{
  "schema": "gpuscale.net/5",
  "engine": 24,                 // sizing engine version
  "studio": "5.14.0",            // UI version that saved the file
  "name": "Project name",
  "projectId": "Project_ab123", // autosave key in the viewer's browser
  "scenarioId": "Project_ab123",// legacy alias, keep equal to projectId
  "mode": "advanced",           // or "normal" (auto-size-as-you-type UX)
  "savedAt": "2026-07-23T09:00:00.000Z",

  // v4-compatible view of the ACTIVE use case + global settings.
  // Older importers read only this; the v5 app reads it for globals.
  "config": {
    "model": {"custom": false, "name": "GPT-OSS 120B"},
    //        or {"custom": true, "params":117,"active":5.1,"hidden":2880,
    //            "layers":36,"kvHeads":8,"headDim":64,"ctx":131072}
    "weightQuant": "FP8",
    "kvQuant": "BF16",
    "preset": "Internal GPT / Copilot",   // CASES name, or null for manual
    "residentSeq": 16384,
    "visibleOut": 800,
    "reasoning": {"mode": "None", "tokens": 0, "extendsKV": true},
    "concurrentCalls": 50,
    "maxBatchPerReplica": 15,
    "kvPolicy": "running",                // "running" | "all"
    "gpu": "H200 141GB NVL",              // GLOBAL: one GPU type per project
    "hardware": {"workers": 1, "gpusPerWorker": 8,
                 "tensorParallel": 2, "resilience": "n"},
    "tuning": {"prefillMFU": 0.5, "decodeMBU": 0.65, "interconnectEff": 0.7,
               "frameworkOverheadMs": 30, "autoSizeUtilPct": 85},
    "sloTargets": {"ttftMs": 1000, "tps": 25, "p95s": 30},
    "estimator": {"sessions": 200, "turnsPerHour": 3, "pctTurnsLLM": 100,
                  "callsPerTurn": 1.5, "burst": 2.5, "callDurS": 9},
    "theme": "light"                      // viewer ignores it on load
  },

  // Full geometry snapshot of the active card - makes the file survive
  // library renames/removals (the app rebuilds a missing model as Custom).
  "snapshot": {
    "model": {"name": "...", "params": 117, "active": 5.1, "hidden": 2880,
              "layers": 36, "kvHeads": 8, "headDim": 64, "ctx": 131072,
              "arch": "MoE (128E/4A, GQA)", "dev": "OpenAI"},
    "gpu": {"name": "H200 141GB NVL", "vram": 141, "bw": 4.8,
            "tflops": 989, "watts": 600, "arch": "Hopper"},
    "weightBytes": 1,           // from weight quant
    "kvBytes": 2                // from KV quant
  },

  // v5: the real project - every use case card.
  "project": {
    "active": 0,                // index of the card open in the editor
    "usecases": [
      {
        "id": "uc1",            // sequential; regenerated by the app anyway
        "name": "Virtual Assistant",
        "isolate": false,       // true = never pool with same-model cards
        "supports": [           // helper models attached to this card
          {"kind": "embed",  "model": "BGE-M3", "on": true},
          {"kind": "rerank", "model": "bge-reranker-v2-m3", "on": true}
        ],
        "concManual": true,
      "sliceU": 0,              // MIG slice units per replica; 0 = dedicated.
                                // Leave 0: Auto-size decides sharing on load     // true = concurrentCalls typed by hand;
                                // false = app re-derives it from estimator
        "activeUsers": 200,     // headcount behind this use case (or null)
        "config":   { /* same keys as top-level config MINUS gpu/tuning/theme;
                         hardware here is only {workers, tensorParallel} */ },
        "snapshot": { /* model + weightBytes + kvBytes only */ }
      }
    ]
  }

  // NOTE: the app's own serialize() also writes "results" and
  // "project.results" blocks. applyConfig() ignores them entirely on import,
  // and the app recomputes everything the moment the link opens - so the
  // encoder omits them on purpose (smaller URLs, nothing to go stale).
}
```

## What the app does when a link opens

1. `applyConfig(payload)` - rebuilds every use-case card, restores globals.
   Unknown model/GPU/quant names fall back gracefully (a missing model is
   rebuilt as Custom from the snapshot geometry; a missing GPU keeps the
   current selection and shows a note).
2. `autoSize(true)` - re-derives concurrency for every card where
   `concManual` is false (Little's law over the estimator fields), then
   re-solves **tensorParallel, workers and maxBatchPerReplica** per pool.

Consequence: the *demand* side (model, quants, preset, tokens, concurrency
or estimator, SLOs, GPU choice, GPUs/worker, resilience, tuning) must be
exact; the *topology* fields are just seeds. Do not interrogate the user
about TP/workers/batch - let the app size them, unless the user explicitly
pins values.

## Concurrency derivation (what `concManual:false` triggers)

```
conc = max(1, ceil(users × turnsPerHour × pctTurnsLLM/100
                   × callsPerTurn × callDurS / 3600 × burst))
```
Presets flagged `direct` (Voice agent, Offline batch) instead set
`conc = users` - callers/jobs ARE the concurrency.

## Pooling semantics (why per-use-case model+quant matters)

Use cases sharing model + weight quant + KV quant + KV policy (and not
`isolate`d) are pooled into one deployment sized for their combined load.
Support models are aggregated per kind+model across cards and packed onto
GPU partitions (MIG/fractional per the GPU's `part` capability).

## Field ranges (the app clamps inputs to these)

residentSeq 128–1,048,576 · visibleOut 16–16,384 · reasoningTokens 0–32,768 ·
concurrentCalls 1–10,000 · maxBatchPerReplica 1–512 · workers 1–64 ·
gpusPerWorker 1–72 · tensorParallel 1–72 · prefillMFU 0.1–0.9 ·
decodeMBU 0.2–0.95 · interconnectEff 0.4–1 · frameworkOverheadMs 0–200.
The encoder clamps too and prints a note whenever it does.
